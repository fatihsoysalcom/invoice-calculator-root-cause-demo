# Invoice Calculator Root Cause Demo

This Python script demonstrates a common scenario where a user might blame a 'tool' (software) for incorrect results, when the actual root cause is a misconfiguration or misunderstanding of how the tool operates. It simulates an invoice calculator where a tax rate is applied, and shows how a perceived 'error' in the total can stem from an unexpected or incorrectly set tax rate, rather than a bug in the calculation logic itself.

## Language

`python`

## How to Run

Save the code as `main.py`. Run from your terminal: `python main.py`
